Что просим выключить

Три переключателя

По каждому: что это, зачем выключать и что мы при этом теряем. Ничего не скрываем — у каждой правки есть цена, хоть и небольшая.

1

Email Address Obfuscation

Security → Settings → Email Address Obfuscation
гипотеза · проверяется за 2 мин
Зачем выключать

Скорее всего именно он ломает кастомайзер. Функция прячет email-адреса от спам-ботов: находит почту в HTML и заменяет ссылкой-заглушкой. Проблема в том, что она лезет и внутрь скриптов — и если в коде попалось что-то похожее на почту, она втыкает туда кусок HTML. Скрипт после этого падает.

Что теряем

Email-адреса на страницах станут видны спам-ботам, как обычный текст. Мелочь, но честно предупреждаем. Если критично — потом закроем это на стороне сайта.

2

Browser Integrity Check

Security → Settings → Browser Integrity Check
блокирует нашу работу
Зачем выключать

Он решает, «настоящий ли браузер» постучался, и отбивает всё остальное страницей 403. Под раздачу попадают наши рабочие инструменты:

  • Не работает WordPress MCP — им мы вносим правки и выгружаем страницы пакетно. Сейчас всё приходится делать руками через браузер.
  • Нельзя проверить трекеры удалённо — то есть подтвердить, что задача по CIPA выполнена, мы не сможем.
Что теряем

Почти ничего. Это грубая проверка заголовков, а не серьёзная защита — основную работу делают WAF и DDoS-защита, их мы не трогаем.

Если выключать нельзя: оставьте включённым, но добавьте правило-исключение для адресов /wp-json/ и /wp-admin/.

3

Web Analytics (RUM)

Speed → Settings → Recommendations → Web analytics
риск по CIPA
Зачем выключать

Это собственный аналитический трекер Cloudflare. Он стреляет до того, как посетитель нажал «Согласен» — ровно то, за что пришла претензия. И его невозможно закрыть баннером согласия (объясняем ниже, почему). Пользы от него нет: аналитика уже собирается через Google Analytics.

Что теряем

Статистику посещаемости внутри панели Cloudflare. Те же цифры есть в GA4, так что потери фактически нет.

4

Мелочи: JS Detections и Schema validation

Security → Settings
по желанию
Зачем выключать

JS Detections подмешивает на страницы ещё один скрипт Cloudflare — лишний в задаче про согласие. Schema validation включена, но настроена в пустую (ноль схем, действие «ничего не делать») — просто мусор в настройках.

Что теряем

Ничего заметного. Это уборка, а не лечение — можно оставить на потом.

Почему это важно именно сейчас

Баннер согласия не может остановить скрипты Cloudflare

Это ключевой момент, который легко упустить. Плагин согласия (Complianz) живёт внутри WordPress. Cloudflare добавляет свои скрипты уже после того, как WordPress отдал страницу — то есть ниже по течению. Плагин физически не видит их и не может заблокировать.

Плагин сделал свою работу, но Cloudflare добавил трекер уже после него. Единственный способ — выключить в панели.
Вывод: пока эти галки включены, задачу по CIPA нельзя закрыть никаким плагином. Это не вопрос настройки сайта — это вопрос настройки Cloudflare.
Отдельная боль

Мы меняем текст или добавляем скрипт — на сайте ничего не появляется. Иногда неделями. Причина простая, и она тоже в Cloudflare.

Чтобы Cloudflare отдал свежую страницу, кэш нужно сбросить. Кнопки для этого у нас сейчас нет.

Просьба

Что нам нужно от вас

Всё, о чём мы просим, — в одном списке. Три блока: что выключить, что прислать и что сфотографировать. Работы в панели — минут пятнадцать.

А

Выключить три переключателя

Обоснование по каждому — в разделе выше. Все обратимы одним кликом.

Б

Прислать доступы

Три пункта. Ни один не даёт доступа к вашему аккаунту, домену или биллингу.

1

API-токен Cloudflare

Чтобы подключить уже установленный плагин и получить кнопку сброса кэша. Global API Key присылать не нужно — он даёт полный доступ ко всему аккаунту. Нужен именно токен, ограниченный одним доменом.

РазделПравоЗачем
Zone → ZoneReadчтобы плагин нашёл нужный домен
Zone → Zone SettingsEditчитать и менять настройки этого домена
Zone → Cache PurgePurgeсобственно сброс кэша
Где создать
My Profile → API Tokens → Create Token → Custom token
В поле Zone Resources выбрать: Include → Specific zone → marketproinc.com
Это безопасно. Таким токеном нельзя тронуть DNS, другие домены, счета или настройки аккаунта. Только кэш и настройки одного этого сайта. Токен можно отозвать в любой момент одной кнопкой.
2

Email аккаунта Cloudflare

Тот адрес, на который заведён аккаунт с доменом marketproinc.com. Плагин может попросить его вместе с токеном.

Сам по себе email ничего не открывает — без токена он бесполезен. Просим на всякий случай, чтобы не гонять переписку по второму кругу.

3

Zone ID

Внутренний номер вашего домена в Cloudflare — строка из 32 символов. Нужен, потому что интерфейс Cloudflare адресует домены по номеру, а не по названию.

Где взять
Выбрать домен marketproinc.com → вкладка Overview → прокрутить вниз, правая колонка, блок API.
Там два поля: Zone ID и Account ID. Нужен первый, у него есть кнопка «Click to copy».

Не секрет: это просто адрес, как номер дома. Без токена по нему ничего сделать нельзя.

В

Прислать три скриншота

Чтобы мы перестали гадать. По убыванию важности — первый ценнее двух остальных вместе.

На присланных ранее скриншотах видна страница Settings — там только карточки настроек. Нам нужны списки правил, а они лежат в соседних разделах.

📍 Где искать: Events и WAF — это не карточки на странице Settings, а отдельные пункты в левом меню, внутри раздела Security. Рядом с пунктом Settings, по которому сделаны первые скриншоты.

Скриншот целиком, со всем списком. Остальное спросим потом, если понадобится.
